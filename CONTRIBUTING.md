# Contributing / 贡献指南

欢迎参与改进 Paper Deconstructor！

---

## 怎么贡献

### 提 Issue
最有价值的反馈来自真实使用体验：

- **哪个阶段卡住了？** Phase 0 的知识地图不准确？Phase 1 解释某个概念太模糊？
- **触发行为不符合预期？** Skill 在不该触发时触发，或该触发时没触发？
- **语气或措辞建议** — Skill 的核心是陪伴感，任何让你觉得"居高临下"的地方都值得改

提 Issue 时请附上：你用的 Claude Code 版本、论文类型（领域）、以及具体发生了什么。

### 提 PR
主要改进方向：

- `SKILL.md` 中的 prompt 措辞优化（更自然、更准确、更有陪伴感）
- 知识盲点协议的资源推荐（补充更好的学习渠道）
- `examples/` 目录中增加新领域的示例会话

**不需要本地开发环境**，只需要 Claude Code 能运行这个 Skill 即可测试。

---

## How to Contribute

### File an Issue
The most valuable feedback comes from real usage:

- **Where did it get stuck?** Inaccurate knowledge map in Phase 0? Vague explanation in Phase 1?
- **Unexpected trigger behavior?** Skill fires when it shouldn't, or doesn't fire when it should?
- **Tone or wording suggestions** — The core of this skill is companionship; anything that feels condescending is worth fixing.

Please include: your Claude Code version, the paper's field, and what specifically happened.

### Submit a PR
Main improvement areas:

- Prompt wording in `SKILL.md` (more natural, accurate, or empathetic)
- Better learning resources in the blind-spot protocol
- New example sessions in `examples/` covering different fields

**No local dev environment needed** — you just need Claude Code to test the Skill.
